After completing of the preprocessing above, please plugged inn your smartfren Haier CE682nya to usb slot, then use the dmesg command, to see the results of the detection of the Linux Debian Squeeze.

And find similar message output look like this:

sr1: scsi-1 drive

sr1, is like a drive such as A: B: C: etc, and is the first detection of as a CD-ROM and scsi-1 in this device. But what you need to note is sr1, because it can change from SR0, SR2, SR3 and so on. Once we know the sr? we can eject your drives with the eject command:

Do not worry if it is not got any message output when we eject The CD-ROM, for that reason we see the results eject command with dmesg command again:

And now it has turned into a flash memory card, if you already attach a SD card on it, you can found your file in this drive. This is just for your information only, what do we really need is a code of vendor and its product, which we will need to connect our Linux Debian Squeeze with our modem. And to get the vendor and its product code we can give a command to the Debian with lsusb command :

From the above output to look for hardware that do not have name such as Textech International Ltd. Mini Optical Mouse and Linux Foundation 1.1 root hub. But if you find more than the hardware without a name, try to unplug other usb device first, to make sure it your modem hardware code.

Once it is certain that Bus 002 Device 005: ID 201e:1022 as one and the only that does not have the hardware name, please note or simply keep in mind that this code section 201e:1022. 201e is vendorcode, and 1022 is product code.

The information we obtain enough already, now our smartfren Haier CE682 can connect with Linux Debian Squeeze as a usb-serial device with modprobe command.

If there is no error message, we can proceed with the wvdialconf command to detect the presence of our modem in the system, and create a basic configuration for the modem:

If there were no problems, then a configuration file with path /etc/wvdial.conf will be made, or if it already there it will be updated. Then we can begin to connect to the internet with wvdial command :
